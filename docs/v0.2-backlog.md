## 1. Graphics and 3D visuals — high priority

The game currently uses procedural Three.js geometry (coloured box and plane meshes) for all ten rooms. Tim has asked for a graphics review and improvement pass.

Work needed:

- Replace or supplement the basic geometry with proper 3D models (GLTF/GLB format) for rooms, furniture, and interactive objects.
- Room atmosphere: lighting, fog, and shadow improvements to support the gothic castle setting.
- The witch's crystal ball (used in cutscenes) needs a visual asset.
- Particle effects or atmospheric magic (spell glows, torch flicker) are listed in the design brief but not yet implemented.
- The design brief references illustrations for the witch character — these have not been produced yet.

Relevant ADRs: `docs/decisions/003-asset-bundling-and-loading.md` (Tier 2 and Tier 3 loading deferred). Jacob's puzzle dependency proposal at `.claude/work/018-sophies-escape-setup/jacob-puzzle-dependency-proposal.md` describes the intended room names and themes.

Exception SE-002 (`docs/exceptions/SE-002-3d-geometry-colours.md`) allows decorative geometry colours to fall outside WCAG contrast. This applies to all visual atmosphere work.

## 5. Architecture conformance — medium priority

Jacob's architecture review identified four conformance gaps (non-blocking for v0.1, due before v1.0):

1. **ADR 002 — intent bus location**: the intent bus is in the wrong module. Should be in `src/core/` not its current location.
2. **ADR 004 — deep freeze**: state objects use shallow freeze (`Object.freeze`) not deep freeze. The reducer should recursively freeze nested objects.
3. **ADR 006 — analytics**: the first room never fires the `room-entered` event. GoatCounter misses the first room visit.
4. **vite.config.js port mismatch**: the development server port does not match the documented default.

---

## 6. Security conditions before v1.0 — medium priority

Jed's security review identified five conditions (non-blocking for v0.1):

1. Remove `'unsafe-inline'` from `script-src` in the CSP (ADR 007 tightening task). Blocked on identifying all inline scripts.
2. Resolve `worker-src blob:` discrepancy — either add it explicitly to the CSP or confirm no worker uses blob URLs.
3. Review service worker scope and cache strategy (ADR 008).
4. Confirm GoatCounter self-hosted `count.js` is the real file, not a placeholder (PR 14 addressed Vite upgrade but the file should be verified).
5. Dependency audit after each major version bump — done for Vite 8, schedule for Three.js and Howler when upgraded.
